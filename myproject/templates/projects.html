{% if language == "English" %}I implemented RSA encryption with 512-bit keys as well as some of the algorithms it uses to do math with very large numbers (division, multiplication, modular exponentiation, and the extended euclidean algorithm) as an exercise. In RSA encryption, a user has a "private" key (number) that only they have access to, and a "public" key (number) that everyone else can access. The application can use a person's private key and an SHA-256 hash (also a number) that's based on the contents of a file to generate a number called a signature. The signature is like a copy of a letter in the user's handwriting: it shows what was originally in the file, and we know the user made it. Another user can later verify that same file hasn't been changed by generating a hash based on the file's contents in the same way, "decrypting" that hash using the first person's "public" key, and comparing the result with the private-key signature made by the first user. If the two values are the same, it's like that whole "letter" has been shown to be in the first user's handwriting--we know no one else added anything to it. This allows a user or application to protect against malicious/unauthorized modifications to the original file. {% else %}Estoy trabajando en cumplir una translación.{% endif %}
