{% if language == "English" %}I implemented RSA encryption with 512-bit keys as well as some of the algorithms it uses to do math with very large numbers (division, multiplication, modular exponentiation, and the extended Euclidean algorithm) as an exercise. In RSA encryption, a user has a "private" key (number) that only they have access to, and a "public" key (number) that everyone else can access. The application can use a person's private key and a hash (also a number) that's based on the contents of a file to generate a number called a signature. The signature is like a copy of a letter in the user's handwriting: it shows what was originally in the file, and we know the user made it. Another user can later verify that same file hasn't been changed. For this, the application generates a hash based on the file's contents in the same way, "decrypts" the signature using the first person's public key, and compares the result with the hash. If the two values are the same, it's like that whole "letter" has been shown to be in the first user's handwriting--we know no one else changed the file. This allows a user or application (with some minor modifications to the project) to protect against malicious/unauthorized modifications to the original file. {% else %}Implementé una aplicación de encriptación RSA con claves de 512 bits, además de unos algoritmos que la encriptación usa para hacer las operaciones aritméticas con números muy largos (como la división, la multiplicación, la exponenciación modular, y el Algoritmo Euclidiano extendido) como un ejercicio. En la encriptación RSA, cada usario tiene una clave privado que solamente el/ella puede obtener y una clave pública que todos pueden usar. La aplicación usa la clave privado de un usario y un valor hash basado en el texto del archivo para generar un número que se llama una firma. La firma es como una carta que es escrita en la letra del usario: indica el contenido original del archivo, y el usario definitivamente lo produjo. Luego, otro usario puede asegurarse de que el archivo no he cambiado. Para esto, la aplicación obtiene un valor hash basado en el texto del archivo como antes, "descifra" la firma usando la clave pública del usario primario, y compara el resultado con el valor hash. Si los números son iguales, sería como si toda esa "carta" fuera escrita en la letra del usario original: sabríamos que nadie más cambiaría el archivo. Esto le permite a un usario o una aplicación (con unos cambios pequeños al proyecto) proteger de modificaciones que son peligrosos o no son autorizados. {% endif %}
